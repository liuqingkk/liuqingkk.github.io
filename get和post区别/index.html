get和post区别

Share
  1. get 从服务器上获取数据，post 是向服务器传送数据
  2. get 提交可以在 url 中看到，post 是通过请求体传递数据，用户看不到
  3. get 传送数据量小(<2kB),post 传送数据量大，一般默认不受限制。实际上，URL 不存在参数上限的问题，HTTP 协议规范没有对 URL 长度进行限制。这个限制是特定的浏览器及服务器对它的限制。IE 对 URL 长度的限制是 2083 字节(2K+35)。对于其他浏览器，如 Netscape、FireFox 等，理论上没有长度限制，其限制取决于操作系统的支持。
  4. get 没有 post 安全，但是执行效率比 post 方法好
