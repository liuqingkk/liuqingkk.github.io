react Hooks

Share
    Hook是react 16.8 新增的特性，它可以让你在不编写class的情况下使用state以及其他的React特性。

    react 已经内置了一些 Hooks，eg：useState、useEffect、useContext、useReducer、useCallback、useMemo、useRef 等 hook。我们主要是将如何利用 Hooks 来进行组件逻辑复用。

   React Hooks 的设计目的，就是加强版函数组件，完全不使用”类”，就能写出一个全功能的组件

    Hook 的含义，React Hooks 的意思是组件尽量写成纯函数，如果需要外部功能和副作用，就用钩子把外部代码“钩”进来，React Hooks 就是那些钩子，所有的钩子都是为函数引入外部功能，所以 React 约定，钩子一律使用 use 作为前缀命名，便于识别

    1. useState()状态钩子,用于引入状态(state)。纯函数不能有状态，所以把状态放在钩子里面，buttonText 是一个变量，指向状态的当前值，第二个成员是一个函数，用来更新状态

2. useContext()共享状态钩子，先有两个组件，希望它们之间是共享状态，示例

    1、使用 React Context API 在组件外部建立一个 Context
const AppContext = React.createContext({})

const Navbar = () => {

    3. useReducer(), action 钩子，React 本身不提供状态管理功能，需要使用外部库(常用 redux),Redux 的核心是组件发出 action 与状态管理器通信，状态管理器收到 action 以后，使用 Reducer 函数算出新的状态，Reducer 函数的形式：(state,action) => newState

useReducer()钩子用来引入 Reducer 功能，

const [state, dispatch] = useReducer(reducer, initialState);

上面是 useReducer()的基本用法，它接受 Reducer 函数和状态的初始值作为参数，返回一个数组。数组的第一个成员是状态的当前值，第二个成员是发送 action 的 dispatch 函数

   4. useEffect() ,副作用钩子，用来引入具有副作用的操作，最常见的就是向服务器请求数据，之前放在 componentDidMount 的代码，现在也可放在 useEffect()，它接受两个参数，参数一：函数，异步操作的代码放在里面；参数二：数组，用于给出 Effect 的依赖项，只要这个数组发生变化就会执行此函数。第二个参数可以省略，这时每次组件渲染时，就会执行 useEffect()。
