三、组件之间的通讯

  1. 父传子：父组件向子组件传值是在调用子组件的时候通过参数(props)传递给子组件
  2. 子传父：有时候子组件需要调用父组件的某个方法，需要在父组件加载子组件的时候将方法当参数传递给子组件,父-代码：
    1. import React, { Component } from 'react'
    2. import CommunicateChild from './communicateChild'
    3.  
    4. export default class Communicate extends Component{
    5.  constructor(props){
    6.  super(props)
    7.  }
    8.  methodToChild(){
    9.  alert("子组件调用父组件的方法")
    10.  }
    11.  render(){
    12.  return(
    13.  <div id="Communicate">
    14.  <h1>父子组件之间的通信</h1>
    15.  <p>{'-----------------------父组件-----------------------'}</p>
    16.  <div>
    17.  <h2>我是父组件</h2>
    18.  </div>
    19.  <br/>
    20.  <p>{'-----------------------子组件-----------------------'}</p>
    21.  <CommunicateChild methodToChild={this.methodToChild.bind(this)}/>
    22.  </div>
    23.  )
    24.  }

    子代码：
    1. import React, { Component } from 'react'
    2.  
    3. export default class CommunicateChild extends Component{
    4.  constructor(props){
    5.  super(props)
    6.  this.getParentMethod = this.getParentMethod.bind(this)
    7.  }
    8.  getParentMethod(){
    9.  this.props.methodToChild()
    10.  }
    11.  render(){
    12.  return(
    13.  <div id="CommunicateChild">
    14.  <div>
    15.  <h3>子组件调用父组件的方法</h3>
    16.  <p><button onClick={this.getParentMethod}>调用父组件的方法</button></p>
    17.  </div>
    18.  </div>
    19.  )
    20.  }
    21. }

Redux 数据流：

https://www.redux.org.cn/docs/basics/DataFlow.html

  1. Redux 是一个单向数据流的开发调试插件
  2. Redux 应用中数据的生命周期遵循下面 4 个步骤：
    1. 调用 store.dispatch(action)。Action 就是一个描述“发生了什么”的普通对象，可以在任何地方调用 store.dispatch(action)，包括组件中、XHR 回调中、甚至定时器中
    2. Redux store 调用传入的 reducer 函数。Store 会把两个参数传入 reducer： 当前的 state 树和 action
    3. 根 reducer 应该把多个子 reducer 输出合并成一个单一的 state 树。
    4. Redux store 保存了根 reducer 返回的完整 state 树。
